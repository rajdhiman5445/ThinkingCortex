// How do Routing works in react? 

Routing is how we map URL path to UI
using react-router-dom

  Key pieces:

  - BrowserRouter: enables routing based on the browser URL
  - Routes: container for route matching
  - Route: defines a path and what component to render
  - Link: client-side navigation without full page reload
  - useNavigate(): navigate in code
  - useParams(): read dynamic route params like /users/:id

May 15

Added a video background to the landing page.

- CSS background-image can use images like svg/png/jpg, but it cannot play a .webm video.
- For video backgrounds, use a real <video> element behind the content.
- The video file is files/images/output.webm.
- The hero text stays above the video using z-index.

Moved the project from Parcel to Vite.

- Parcel had trouble handling the .webm asset cleanly.
- Vite works better for this React app and can import video assets directly.
- The main entry file is now index.jsx instead of index.js.
- Components with JSX were renamed from .js to .jsx.
- Vite config lives in vite.config.js.

Useful commands:

- npm run dev: start local Vite dev server
- npm run build: create production build
- npm run preview: preview production build locally

Background video import in Vite:

import heroVideo from "./files/images/output.webm";

Then use it in the video source:

<source src={heroVideo} type="video/webm" />
